# HAR to Spreadsheet API Extractor 🚀

## Overview
This Python project automates the extraction of API calls, CRUD operations, Tenant IDs, and module data from `.har` (HTTP Archive) files into a structured spreadsheet. 

I built this tool to replace a highly repetitive manual data entry process, applying a "smart work over hard work" approach to streamline testing workflows and save significant operational time.

## The Scenario & Problem
During routine website testing and documentation, I was tasked with manually clicking through an application, inspecting the network tab, and recording every API call, CRUD operation, and Tenant ID into a spreadsheet. 

* **The Scale:** Approximately 2,000 to 3,000 individual calls to document.
* **The Time Sink:** Estimated 1 to 2 weeks of full-time, manual data entry.
* **The Risk:** High probability of human error due to the tedious nature of copying and pasting network responses.

## The Solution
Instead of spending weeks manually recording clicks and responses, I developed a Python script to automate the entire extraction process. By exporting the network activity as a `.har` file, this script parses the JSON structure and automatically formats the required data points into a clean, ready-to-use spreadsheet.

### Key Features
* **Automated Parsing:** Reads standard `.har` files generated by any modern browser.
* **Targeted Extraction:** Isolates and extracts:
  * API Endpoints
  * CRUD Operations (GET, POST, PUT, DELETE)
  * Payload Data
  * Module and Sub-module Calls
* **Spreadsheet Generation:** Outputs the parsed data directly into a clean `.csv` format ready for Google Sheets or Excel.

## Impact
* **Time Saved:** Reduced a 2-week manual task down to the few seconds it takes to run the script.
* **Accuracy:** Eliminated the risk of human error from manual copy-pasting.
* **Efficiency:** Demonstrated the value of automation in testing pipelines.

## Technologies Used
* Python
* `json` (for parsing the HAR file)
* `csv` (for spreadsheet generation)
* `urllib.parse` (for routing and module extraction)

## How to Use
1. Clone the repository: `git clone [your-repo-link]`
2. Ensure you have Python installed on your system.
3. Export your browser's network activity as a `.har` file. (A `sample_data.har` is provided for testing).
4. Update the `HAR_FILE_PATH` variable inside the script to match your file's name.
5. Run the script: `python paycheq.py`
6. The extracted data will be instantly saved to `api_mapping.csv` in the same directory.
